# 《悬观谜案：百年轮回》动态叙事系统设计

## 一、动态场景描述系统

### 1.1 场景描述变体生成规则

每个场景的描述由多个维度组合生成：
- **时间维度**：白天/黄昏/夜晚/深夜
- **进度维度**：初期/发现线索后/解开谜题后/真相揭示后
- **状态维度**：理智高低/体温高低/体力高低
- **事件维度**：关键事件触发前/触发后

### 1.2 悬云观山门场景描述变体

#### 变体1：初期调查（白天，理智正常，体温正常）
```
暴雪中的悬云观山门。石质门框在风雪中若隐若现，门楣上"悬云观"三字匾额被积雪半掩，木质边缘的朱漆已褪成暗红，像干涸的血迹。山门两侧石狮头顶积雪，如戴白帽，但左侧石狮底座有新鲜摩擦痕迹，与周围积雪形成鲜明对比。门内庭院积雪已深，无人足迹。远处三清殿轮廓在雪幕中浮动，如海市蜃楼。
```

#### 变体2：发现线索后（黄昏，理智中等，体温偏低）
```
暮色中的悬云观山门。风雪稍歇，但寒意更甚。你注意到石狮底座的摩擦痕迹延伸向门内，似乎有人反复拖动重物。匾额上的"悬云观"三字在暮光中投下细长阴影，那"悬"字的最后一笔似乎指向右侧屋檐。庭院积雪上隐约可见几道不规则的凹陷，像是有人曾在此徘徊。三清殿的轮廓在渐暗的天色中显得更加沉重，仿佛一头蛰伏的巨兽。
```

#### 变体3：解开谜题后（夜晚，理智较低，体温低）
```
夜色中的悬云观山门。风雪再起，但此刻你看到的景象已不同。石狮底座的新鲜痕迹在你眼中连接成某种图案——那是墨家机关术中的"悬权"标记。匾额上的字迹在月光下仿佛在流动，"悬云观"三字时隐时现，如同呼吸。庭院积雪下的凹陷现在清晰可见，它们组成一个巨大的八卦阵图，中心指向三清殿。远处殿宇的轮廓不再模糊，你能看到屋檐下悬挂的铜铃在无风自动，发出微不可闻的叮当声。
```

#### 变体4：真相揭示后（深夜，理智临界，体温危险）
```
死寂中的悬云观山门。暴雪已停，但寒意刺骨。石狮的眼睛在月光下泛着诡异的红光，仿佛活了过来。匾额上的字迹完全消失，只剩一片墨黑，那黑色如活物般蠕动，缓慢渗出匾额边缘。庭院积雪下的八卦阵图发出微弱蓝光，中心凹陷处升起淡淡雾气，雾气中隐约有人形轮廓。三清殿不再遥远——它就在你眼前，殿门半开，门缝中透出的不是烛光，而是深不见底的黑暗，那黑暗正缓缓向你蔓延。
```

### 1.3 三清殿场景描述变体

#### 变体1：白天正常状态
```
三清殿内光线昏暗，只有几缕天光从高窗透入，在布满灰尘的地面上投下斑驳光影。三尊神像端坐高台，金身剥落处露出其下泥土本质，眼神在阴影中仿佛在注视着你。供桌上香炉已冷，香灰散落如骨灰。殿内弥漫着陈年檀香与霉味混合的气息。你注意到地面向西北方向轻微倾斜，墙角有暗红色水渍痕迹。
```

#### 变体2：夜晚低理智状态
```
三清殿内烛火摇曳，投下的影子比实物更长、更扭曲。三尊神像的面容在光影中不断变化——时而悲悯，时而狰狞。供桌上的香炉不知何时重新点燃，但冒出的不是青烟，而是墨汁般的黑雾。地面倾斜更加明显，你几乎要扶住柱子才能站稳。墙角的水渍痕迹在烛光下反射暗红光泽，像未干的血迹。殿内响起若有若无的诵经声，但仔细听时又消失不见。
```

#### 变体3：体温过低状态
```
三清殿内寒冷刺骨，比户外更甚。你的呼吸在空气中凝成白雾，但神像周围却没有任何雾气——它们仿佛在吸收热量。地面结了一层薄霜，霜花在倾斜的地面上形成诡异图案，像是某种文字。供桌上的香灰结冰，形成尖锐的冰晶。墙角的水渍已冻结成暗红色冰柱。殿内异常安静，连自己的心跳声都变得遥远，仿佛被这寒冷吞噬。
```

#### 变体4：解开地板机关后
```
三清殿地面机关已启动。原本倾斜的地板现在水平如镜，但镜面之下隐约可见复杂齿轮结构缓慢转动。三尊神像的位置微妙变化，现在它们的手指共同指向殿后墙壁某处。供桌下沉半尺，露出下方暗格，暗格中放置着一卷泛黄经卷。墙角水渍痕迹消失，取而代之的是墙壁上浮现的墨迹文字，文字随齿轮转动而缓慢变化。殿内温度回升，但空气中多了一股铁锈与机油混合的金属气味。
```

### 1.4 藏经阁场景描述变体

#### 变体1：初次进入（理智正常）
```
藏经阁内书架林立，直抵屋顶。大部分经书已霉变虫蛀，纸页粘连成块。空气中弥漫着纸张腐烂的甜腻气味。你注意到西北角书架有明显空缺，像是近期有人取走过大量书籍。地板上有拖拽痕迹，痕迹尽头是一面看似普通的墙壁，但墙壁与地板接缝处有新鲜木屑。
```

#### 变体2：发现暗门后（理智中等）
```
藏经阁的书架现在在你眼中排列成某种阵法。西北角的空缺位置恰好对应北斗七星中的天枢星位。地板上的拖拽痕迹组成一个箭头，直指那面墙壁。墙壁接缝处的木屑在烛光下反射微光——那是金属摩擦木头的痕迹。你听到墙壁后传来极其微弱的机械转动声，规律如心跳。
```

#### 变体3：暗门开启时（理智低）
```
暗门开启的瞬间，藏经阁内所有书架同时震动，经书如落叶般飘落。墙壁后的机械声骤然放大，如巨兽苏醒。暗门内透出的不是烛光，而是幽蓝冷光，那光芒在空气中留下残影，仿佛时间在此处变慢。地板上的拖拽痕迹开始发光，组成一行文字："机关不言，其理自现"。空气温度骤降，你的呼吸在面前凝成冰晶。
```

#### 变体4：密室探索后（理智临界）
```
从密室返回藏经阁，一切已不同。书架上的经书全部消失，只剩空荡荡的木架。墙壁上浮现出血红色文字，文字随你的视线移动而变换。地板中央出现一个深不见底的洞口，洞内传出齿轮咬合的金属摩擦声，那声音中夹杂着人声低语。暗门自动关闭，门缝中渗出墨黑色液体，液体在地面蔓延，所到之处木板腐朽如百年。
```

## 二、事件防重复叙事处理

### 2.1 重复检查已调查对象

#### 石狮检查（第一次）
```
你仔细检查左侧石狮。明代石雕，风化严重，但底座有明显新鲜摩擦痕迹，痕迹深度约2毫米，方向由外向内。石狮右前爪下方压着一片褪色红布，布上绣有模糊八卦图案。
```

#### 石狮检查（第二次）
```
再次检查石狮，没有新发现。摩擦痕迹依然清晰，红布仍在原处。风雪似乎更大了，石狮头顶积雪增厚，仿佛戴上了更重的白帽。
```

#### 石狮检查（第三次）
```
你反复检查石狮，开始怀疑自己是否遗漏了什么。就在这时，一阵强风吹过，石狮右眼处的积雪滑落——你注意到石眼瞳孔位置有一个极小的孔洞，直径不足1毫米，像是锁孔。
```

#### 石狮检查（第四次及以上）
```
（理智>60）过度检查浪费时间。石狮不会告诉你更多秘密，或许该寻找其他线索。
（理智<40）石狮的眼睛似乎在转动，跟随你的移动。你感到一阵眩晕，耳边响起石质摩擦的刺耳声音。
```

### 2.2 重复与NPC对话

#### 玄真道长对话（第一次询问历史）
```
玄真道长："悬云观建于明末，开山祖师号'悬机子'，精通道教术数与墨家机关。观内建筑多有机关暗室，是为防匪防盗。"
```

#### 重复询问历史
```
玄真道长：（略显不耐）"贫道已说过，观建于明末。林施主若对建筑感兴趣，可自行观察，莫要反复询问同样问题。"
```

#### 第三次询问历史
```
玄真道长：（面色不悦）"林施主，有些事知道太多并非好事。道观历史就是道观历史，无需深究。"
```

```
玄真道长：（突然暴怒）"够了！你若再问，就请离开悬云观！"（道长拂袖而去，今日不再接受对话）
```

### 2.3 重复触发机关

#### 地板机关第一次触发
```
你对倾斜地板使用水平仪。气泡偏移显示2度倾斜，方向西北。敲击西北墙角，回声空洞，说明墙后有空间。计算重量分布，需要约50kg变化才能触发机关。
```

#### 重复触发（未满足条件）
```
再次测量，结果相同。机关需要特定条件才能启动，单纯测量无法触发。
```

#### 重复触发（满足条件后）
```
你将重物放置在计算位置。地板发出低沉轰鸣，缓慢恢复水平。下方齿轮转动声清晰可闻，墙壁某处暗格弹开。
```

#### 重复触发（已解决后）
```
机关已启动，地板保持水平。齿轮声持续但稳定，暗格保持开启状态。重复操作无新效果。
```

## 三、状态变化的叙事反馈

### 3.1 体力下降的渐进描述

#### 体力80-100%（充沛）
```
你感觉精力充沛，可以轻松应对各种挑战。移动迅速，思考清晰。
```

#### 体力50-79%（正常）
```
稍有疲惫，但不影响行动。长时间站立后腿部微酸。
```

#### 体力30-49%（疲劳）
```
疲劳感明显。移动速度减慢，呼吸稍显急促。需要时不时停下来休息。
```

#### 体力10-29%（严重疲劳）
```
每一步都感到沉重。视线偶尔模糊，需要扶墙才能站稳。思考变得困难，简单问题也需要更多时间。
```

#### 体力0-9%（濒临昏厥）
```
世界在旋转。你几乎无法站立，只能跪倒在地。耳边嗡嗡作响，视线中只有黑白两色。随时可能失去意识。
```

### 3.2 理智变化的叙事效果

#### 理智80-100%（稳定）
```
思维清晰，逻辑严谨。能够冷静分析线索，不受环境影响。
```

#### 理智50-79%（动摇）
```
开始感到不安。眼角余光似乎看到阴影移动，但转头时又消失。需要集中精神才能保持冷静。
```

#### 理智30-49%（混乱）
```
幻觉频繁出现。墙壁上的污渍看起来像人脸，风声像是低语。难以区分现实与想象，需要反复确认所见是否真实。
```

#### 理智10-29%（濒临崩溃）
```
现实扭曲。时间忽快忽慢，空间伸缩变形。听到不存在的声音，看到不存在的景象。开始怀疑自己的记忆和判断。
```

#### 理智0-9%（崩溃）
```
完全失去现实感。你无法确定什么是真，什么是假。可能看到已死之人对你说话，或者自己身处完全不同的时空。游戏难度极大增加，可能触发强制坏结局。
```

### 3.3 体温变化的生理反馈

#### 体温36-38℃（正常）
```
体温正常，身体机能良好。能够有效抵抗寒冷。
```

#### 体温34-35.9℃（轻度失温）
```
开始发抖，手指麻木。呼吸变浅，心跳加快以维持热量。体力恢复速度降低20%。
```

#### 体温32-33.9℃（中度失温）
```
剧烈颤抖无法控制。思维迟钝，说话困难。面部苍白，嘴唇发紫。动作迟缓，解谜时间增加50%。
```

#### 体温28-31.9℃（重度失温）
```
颤抖停止——这是危险信号。意识模糊，产生温暖幻觉（反而觉得热）。可能做出危险决定（如脱掉衣服）。生命危险，需要立即取暖。
```

#### 体温<28℃（濒死）
```
意识丧失边缘。心跳极慢，呼吸微弱。如不立即获救，将在游戏时间内死亡。
```

### 3.4 状态组合的特殊叙事

#### 低理智+低体温（最危险组合）
```
寒冷与疯狂交织。你既感到刺骨寒意，又产生温暖幻觉。墙壁上的冰霜看起来像盛开的花朵，你想要触摸它们。同时，你听到温暖的声音呼唤你走向更冷的地方。这是最危险的时刻，可能做出致命选择。
```

#### 低体力+低理智（虚弱混乱）
```
身体虚弱，精神恍惚。你分不清是身体疲惫导致幻觉，还是幻觉导致身体疲惫。简单的动作需要巨大努力，而努力的成果可能只是另一个幻觉。
```

#### 正常状态组合（最佳状态）
```
身心平衡。你能够冷静分析环境，有效利用体力，保持适当体温。这是解决问题的最佳状态。
```

## 四、环境互动的状态反馈

### 4.1 与寒冷环境互动

#### 户外暴雪中（体温正常时）
```
暴雪扑面，寒意刺骨。你拉紧衣领，但寒风仍从缝隙钻入。需要尽快找到避寒处。
```

#### 户外暴雪中（体温低时）
```
暴雪不再是雪，而是无数冰针。每一片雪花都像在吸取你的体温。你感到寒冷从皮肤渗透到骨髓，思考变得极其困难。
```

#### 靠近火炉（体温恢复）
```
火炉散发出令人安心的温暖。热量逐渐驱散寒意，僵硬的手指开始恢复知觉。理智值轻微恢复（+5）。
```

#### 长时间取暖后
```
过度取暖导致昏昏欲睡。火炉的热量让你放松警惕，可能错过重要线索或危险信号。
```

### 4.2 与黑暗环境互动

#### 进入黑暗房间（理智正常）
```
房间一片漆黑。你摸索着寻找光源，眼睛逐渐适应黑暗，能勉强分辨轮廓。
```

#### 进入黑暗房间（理智低）
```
黑暗如实体般压迫而来。你感觉有无数眼睛在黑暗中注视你，呼吸声在房间内回响放大。需要立即点亮光源，否则理智将持续下降。
```

#### 点燃蜡烛后
```
烛光驱散黑暗，但只照亮有限范围。光线边缘的阴影似乎在蠕动，烛火摇曳时影子也跟着扭曲。
```

#### 手电筒光束
```
手电筒光束刺破黑暗，但光束外的黑暗更加深邃。光束照到的地方，细节清晰可见；光束未及之处，仿佛不存在。
```

### 4.3 与诡异现象互动

#### 听到机械声（第一次）
```
深夜，你听到藏经阁方向传来规律的机械转动声，像是巨大齿轮咬合。声音持续约30秒后停止。
```

#### 听到机械声（理智低时）
```
机械声再次响起，但这次声音中夹杂着人声低语。你听不清具体内容，但那语调充满痛苦与绝望。声音似乎来自四面八方，又似乎就在耳边。
```

#### 调查声音来源
```
你循声前往藏经阁。越靠近，机械声越清晰，但人声低语却消失了。到达藏经阁时，声音恰好停止，仿佛在等你到来。
```

#### 记录声音规律
```
你使用笔记本记录机械声规律：每2小时17分响起，持续33秒。声音频率呈正弦波变化，峰值在中间时刻。这显然是精心设计的机关，而非自然现象。
```

## 五、叙事逻辑与玩家引导

### 5.1 渐进式信息揭示

### 7.2 动态描述生成算法

```javascript
function generateSceneDescription(locationId, gameState) {
  const scene = scenes[locationId];
  let description = scene.base_description;

  // 应用时间变体
  if (scene.variants.time[gameState.time]) {
    description = scene.variants.time[gameState.time].description;
  }

  // 应用进度变体（按优先级）
  const progressVariants = Object.entries(scene.variants.progress)
    .filter(([key, variant]) => checkRequirements(variant.requirements, gameState))
    .sort((a, b) => b[1].priority - a[1].priority); // 优先级高的覆盖低的

  if (progressVariants.length > 0) {
    description = progressVariants[0][1].description;
  }

  // 应用状态变体
  const stateVariants = Object.entries(scene.variants.state)
    .filter(([key, variant]) => checkStateRequirements(variant.requirements, gameState.player))
    .sort((a, b) => b[1].priority - a[1].priority);

  if (stateVariants.length > 0) {
    // 状态变体可以叠加或替换，取决于设计
    description = applyStateVariant(description, stateVariants[0][1]);
  }

  // 应用特殊事件标记
  if (gameState.flags[`${locationId}_special_event`]) {
    description = applySpecialEvent(description, locationId);
  }

  return description;
}

function checkStateRequirements(requirements, playerState) {
  if (requirements.sanity) {
    const [op, value] = parseCondition(requirements.sanity);
    return compare(playerState.sanity, op, value);
  }
  if (requirements.temperature) {
    const [op, value] = parseCondition(requirements.temperature);
    return compare(playerState.temperature, op, value);
  }
  if (requirements.stamina) {
    const [op, value] = parseCondition(requirements.stamina);
    return compare(playerState.stamina, op, value);
  }
  return true;
}
```

### 7.3 重复事件计数器

```javascript
class EventCounter {
  constructor() {
    this.counts = new Map(); // eventId -> count
    this.lastEventTime = new Map(); // eventId -> timestamp
  }

  recordEvent(eventId) {
    const count = this.counts.get(eventId) || 0;
    this.counts.set(eventId, count + 1);
    this.lastEventTime.set(eventId, Date.now());

    // 根据计数返回相应的描述
    return this.getEventDescription(eventId, count + 1);
  }

  getEventDescription(eventId, count) {
    const handler = repeatEventHandlers[eventId];
    if (!handler) return handler.first.description;

    if (count === 1) return handler.first.description;
    if (count === 2) return handler.second.description;
    if (count === 3) return handler.third.description;
    if (count >= 4) {
      // 第四次及以后，可能随机选择不同反馈
      const variants = handler.fourth_plus.variants || [handler.fourth_plus];
      const index = Math.floor(Math.random() * variants.length);
      return variants[index].description;
    }
  }

  // 冷却时间检查（防止快速重复点击）
  checkCooldown(eventId, cooldownMs = 5000) {
    const lastTime = this.lastEventTime.get(eventId);
    if (!lastTime) return true;
    return Date.now() - lastTime > cooldownMs;
  }
}
```
