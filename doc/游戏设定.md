# 《悬观谜案：百年轮回》文字交互式解密游戏系统设计

## 一、游戏概述

### 游戏类型
基于文本的交互式推理解密游戏，玩家扮演主角林墨（土木工程专业学生），在暴雪困观的悬云观中调查命案、破解机关、揭开百年秘密。

### 核心玩法
- **文字探索**：阅读环境描述，发现线索
- **交互解谜**：点击高亮文字检查/收集/使用物品
- **道具管理**：收集和使用道具解决机关谜题
- **任务推进**：完成关键动作解锁新区域和剧情
- **对话选择**：与NPC对话获取信息和线索

### 技术架构
- **前端**：Web-based文字游戏界面（HTML/CSS/JavaScript）
- **后端**：游戏状态管理、逻辑处理（可选Node.js/Python）
- **数据存储**：JSON格式的游戏数据（场景、道具、任务、对话）

#### 交互功能

2. **道具使用**：

   - 最大携带数量
   - 部分道具不可丢弃（关键剧情物品）

### 4.5 整体设计风格
核心美学总纲：褪色的红与游走的墨
核心概念：“被遗忘的敬仰，与异化的日常”
本作的视觉风格并非单纯的阴森，而是一种渗入骨髓的荒芜感与错位感。它源于对传统民俗美学的解构与异化。

#### 视觉基调：

色彩核心：“褪色红”、“骨白”与“墨黑”。红色不再喜庆，而是像褪色的春联、干涸的血迹或生锈的铁锈；白色是枯骨的哑光白、纸钱的惨白或雾气的浑浊白；黑色则是深不见底的阴影、墨汁般流动的污浊。

质感特征：“腐烂的华丽”。无论是雕梁画栋还是神像雕塑，都处于一种过度腐朽、虫蛀、霉变的状态。华丽的细节正在被时间吞噬，露出其下粗糙、狰狞的本质。

光影氛围：“游走的墨”。光线是有“重量”和“毒性”的。光源（如灯笼、烛火）的光晕会像在水中一样波动，阴影并非静止，而是像有生命般缓慢蠕动、渗透，仿佛墨汁滴入清水，将黑暗扩散至每一个角落。

####  场景设计指导：城市与山区的变奏
##### 一、 城市景观：异化的日常
城市是“日常”被扭曲的舞台，恐怖感来源于熟悉事物的陌生化。

- 建筑与布局：

老城区/城中村：密集、潮湿、逼仄的巷弄，电线如蛛网般杂乱交织，像一张捕获生魂的网。屋檐低垂，几乎要压到头顶，墙角长满因吸收污秽而变异的暗红色青苔。

标志物：路口常见的老榕树，气根垂如帘幕，树后可能藏着神龛，香火已断，只剩黑色香脚。一座孤零零的、翻新到一半就停工的老宅，一半是现代瓷砖，一半是斑驳的木板，交界处仿佛时空的裂痕。

店铺与场所：扎纸店橱窗里的纸人，面容不再是标准微笑，而是带着一种诡异的、与路过者对视角度的凝视。空置的戏台，幕布在无风中自动微动，仿佛还有看不见的戏班在上演。

- 装饰与细节：

符咒与痕迹：门上贴着的不是对联，而是层层叠叠、墨迹已模糊的符纸。地上有烧纸钱后留下的黑色印记，形状扭曲，像痛苦的人脸。

异化的传统元素：石狮子的眼睛被红漆涂抹，像在流血泪。屋脊上的神兽（仙人走兽）面容扭曲，仿佛在承受巨大的痛苦。红灯笼透出的光是病态的橘黄，在地面投下的影子比实物更长、更扭曲。

##### 二、 山区景观：荒芜的敬仰
山区是“信仰”被遗弃的荒野，恐怖感来源于神圣事物的堕落。

- 自然与人文的交融：

植被：树木姿态狰狞，如张牙舞爪的鬼魅。竹林在风中摩擦，发出类似人低语的“沙沙”声。藤蔓如蛇般缠绕着所有废弃的建筑，将它们拉回大地。

地形：山间小径被疯长的野草淹没，路旁每隔一段就有残破的路碑或指路碑，上面名字被风雨侵蚀，或被苔藓覆盖。

- 建筑与遗迹：

废弃庙宇：庙门半开，里面一片漆黑。门槛上散落着被撕碎的经幡。殿内神像的金身剥落，露出其下泥土或木头的本质，眼神在黑暗中却仿佛在注视着你。供桌上没有供品，只有干涸成一团的黑色不明物和老鼠的残骸。

义庄/祠堂：建筑结构极不对称，窗户开得奇高且小，像紧闭的眼睛。墙根有被水长期浸泡的痕迹，长出诡异腥红的菌类。门板上的门神画像已面目模糊，但总感觉它们在用手势示意你“离开”。

坟墓与祭祀：散落在山间的无主孤坟，墓碑东倒西歪，坟头被野狗刨开黑洞。偶尔可见一棵孤树上挂着褪色的红布条，树下的土地公祠被泥土掩埋过半，只露出石制的、风化模糊的香炉。

#### 确保风格统一的关键元素
- 通用元素库（素材复用）：

纸张：破碎的符纸、褪色的红纸、被泥水浸透的黄纸钱，在城市角落和山间小径都会出现。

绳结与缠绕：无论是城市的电线还是山区的藤蔓，都以扭曲缠绕的形态出现，象征着束缚与纠缠。

特定纹理：设定统一的“腐化纹理”库，如霉斑、水渍、虫蛀痕迹、锈蚀金属，在不同场景以相似逻辑应用。

- 氛围连续性：

无论在何处，声音设计都应包含低频的嗡鸣（类似诵经或耳鸣）、远处若有若无的锣钹声、以及风吹过缝隙的呜咽。

“雾气”统一设计为具有“记忆”的、略带粘稠感的粒子，它在城市地面贴地滚动，在山谷中则缓慢沉积。

- 光线法则：

建立统一的光源系统。例如：红色光源（灯笼、香火）永远无法照亮周围的黑暗，反而会被黑暗“侵蚀”，使其光晕边缘呈现不规则的黑丝。唯一纯净的光源（如月光、玩家的手电）会暴露更多扭曲现实的细节，带来更大的危险。

通过以上设计，无论是繁华旧城区的压抑，还是荒野山村的荒凉，都将统一于“褪色的红与游走的墨”这一核心美学之下，共同编织一个让人脊背发凉又深深着迷的中式恐怖世界。

#### UI风格设定

- 布局和尺寸为竖屏显示，不需要支持横屏和pad大屏
- 各类窗口/容器，需要区分标题，边框，内容区等分区设计，分别有独立的底纹,动效，暗角，渐变等效果

#### 与现有系统的整合

1. **与道具系统整合**
   - 某些决策需要特定道具才能选择
   - 决策结果可能获得/失去关键道具
   - 道具使用方式影响决策成功率

2. **与任务系统整合**
   - 决策触发新任务或改变任务目标
   - 任务完成方式受历史决策影响
   - 决策解锁/关闭任务链

3. **与对话系统整合**
   - 决策影响NPC关系和对话选项
   - 历史决策成为对话话题
   - 决策风格影响对话语气选项

4. **与状态系统整合**
   - 决策消耗/恢复各种状态值
   - 状态阈值限制决策选项
   - 决策结果产生永久状态改变

#### 实现技术要点

```javascript
// 决策管理器类
class DecisionManager {
  constructor() {
    this.activeDecision = null;
    this.decisionHistory = [];
    this.echoEffects = new Map(); // 决策回响效果
  }

  // 触发决策
  triggerDecision(decisionId, context) {
    const decision = this.getDecision(decisionId);
    if (!this.checkRequirements(decision, context)) {
      return false; // 不满足条件，不触发
    }

    this.activeDecision = decision;
    this.showDecisionDialog(decision, context);
    return true;
  }

  // 处理决策选择
  handleChoice(optionId) {
    const option = this.activeDecision.options[optionId];
    const outcome = this.calculateOutcome(option);

    // 应用即时效果
    this.applyImmediateEffects(option.immediate_effects, outcome);

    // 记录决策历史
    this.recordDecision(option, outcome);

    // 设置回响效果
    this.setEchoEffects(option.long_term_consequences);

    // 触发后续事件
    this.triggerFollowupEvents(option, outcome);

    return outcome;
  }

  // 检查决策回响
  checkDecisionEcho(location, npc, action) {
    const echoes = this.echoEffects.get(location) || [];
    echoes.forEach(echo => {
      if (echo.condition(location, npc, action)) {
        echo.effect(); // 应用回响效果
      }
    });
  }
}
```

```javascript
function generateDynamicDescription(sceneData, gameState) {
  let description = sceneData.base_description;
  const applicableVariants = [];

  // 收集适用变体
  // 1. 时间变体
  if (sceneData.variants.time[gameState.time_period]) {
    applicableVariants.push(sceneData.variants.time[gameState.time_period]);
  }

  // 2. 进度变体（按优先级）
  Object.values(sceneData.variants.progress).forEach(variant => {
    if (checkRequirements(variant.requirements, gameState)) {
      applicableVariants.push(variant);
    }
  });

  // 3. 状态变体
  Object.values(sceneData.variants.state).forEach(variant => {
    if (checkStateRequirements(variant.requirements, gameState.player)) {
      applicableVariants.push(variant);
    }
  });

  // 选择最高优先级变体
  if (applicableVariants.length > 0) {
    applicableVariants.sort((a, b) => (b.priority || 1) - (a.priority || 1));
    description = applicableVariants[0].description;

    // 组合高优先级变体（可选）
    const highPriority = applicableVariants.filter(v => (v.priority || 1) >= 3);
    if (highPriority.length > 1) {
      description = combineVariants(highPriority);
    }
  }

  return description;
}
```

### 11.3 事件防重复机制

#### 11.3.1 设计原则

1. **一次性事件**：关键剧情事件只能触发一次
2. **渐进反馈**：重复操作提供不同反馈，避免单调
3. **状态影响**：玩家状态影响重复操作结果
4. **冷却机制**：防止快速重复点击滥用系统

#### 11.3.2 事件状态追踪

```javascript
class EventManager {
  constructor() {
    this.events = new Map(); // eventId -> {triggerCount, lastTriggerTime, state}
    this.eventStates = new Map(); // eventId -> "available"|"triggered"|"blocked"|"cooldown"
  }

  // 检查事件是否可触发
  canTriggerEvent(eventId, context = {}) {
    const event = this.events.get(eventId);
    if (!event) return { canTrigger: true, reason: "first_time" };

    // 检查事件状态
    switch (event.state) {
      case "triggered":
        return { canTrigger: false, reason: "already_triggered" };
      case "blocked":
        return { canTrigger: false, reason: "blocked_by_conditions" };
      case "cooldown":
        const cooldownRemaining = event.cooldownEnd - Date.now();
        if (cooldownRemaining > 0) {
          return { canTrigger: false, reason: "cooldown", remaining: cooldownRemaining };
        }
        break;
    }

    // 检查前置条件
    if (event.prerequisites) {
      const unmet = event.prerequisites.filter(prereq =>
        !this.checkCondition(prereq, context)
      );
      if (unmet.length > 0) {
        return { canTrigger: false, reason: "prerequisites_not_met", unmet };
      }
    }

    return { canTrigger: true, reason: "available" };
  }

  // 触发事件
  triggerEvent(eventId, context = {}) {
    const canTrigger = this.canTriggerEvent(eventId, context);
    if (!canTrigger.canTrigger) {
      return { success: false, reason: canTrigger.reason };
    }

    // 记录触发
    const event = this.events.get(eventId) || { triggerCount: 0 };
    event.triggerCount++;
    event.lastTriggerTime = Date.now();
    event.state = "triggered";
    this.events.set(eventId, event);

    // 设置冷却时间（如果配置）
    if (event.cooldown) {
      event.state = "cooldown";
      event.cooldownEnd = Date.now() + event.cooldown;
    }

    return { success: true, triggerCount: event.triggerCount };
  }
}
```

```javascript
function generateStatusChangeDescription(oldState, newState, changes) {
  const descriptions = [];

  // 理智变化描述
  if (changes.sanity) {
    if (changes.sanity < -10) descriptions.push("强烈的精神冲击让你头晕目眩。");
    else if (changes.sanity < -5) descriptions.push("不安感如潮水般涌来。");
    else if (changes.sanity < 0) descriptions.push("你感到一丝不安。");
    else if (changes.sanity > 5) descriptions.push("心神逐渐安定下来。");
  }

  // 体温变化描述
  if (changes.temperature) {
    if (changes.temperature < -0.3) descriptions.push("刺骨寒意瞬间穿透衣物。");
    else if (changes.temperature < -0.1) descriptions.push("寒冷逐渐加剧。");
    else if (changes.temperature > 0.5) descriptions.push("温暖包裹全身，僵硬的手指逐渐恢复知觉。");
  }

  // 体力变化描述
  if (changes.stamina) {
    if (changes.stamina < -20) descriptions.push("极度疲惫袭来，几乎站立不稳。");
    else if (changes.stamina < -10) descriptions.push("疲惫感明显加重。");
    else if (changes.stamina > 20) descriptions.push("精力充沛，疲劳一扫而空。");
  }

  // 组合效果描述
  if (newState.sanity < 40 && newState.temperature < 35) {
    descriptions.push("寒冷与疯狂交织，现实开始扭曲。");
  }

  return descriptions.length > 0 ? descriptions.join(" ") : "";
}
```

```javascript
class EnhancedGameLoop {
  update() {
    // 1. 更新时间，触发时间相关状态变化
    this.updateTimeAndEnvironment();

    // 2. 更新自然状态变化（环境、时间影响）
    this.updateNaturalStatusChanges();

    // 3. 处理玩家交互
    this.handlePlayerInteractions();

    // 4. 生成动态场景描述
    this.generateDynamicSceneDescriptions();

    // 5. 检查并触发阈值事件
    this.checkStatusThresholdEvents();

    // 6. 渲染更新
    this.renderAll();
  }

  handlePlayerInteractions() {
    // 使用事件防重复系统
    const result = this.eventManager.processInteraction(
      interactionId,
      playerAction,
      context
    );

    if (result.success) {
      // 应用状态变化
      this.statusManager.applyChanges(result.effects);

      // 生成叙事反馈
      const feedback = this.narrativeGenerator.generateFeedback(
        result,
        this.playerState
      );

      // 更新UI
      this.ui.displayFeedback(feedback);
    }
  }
}
```
