## 技术实现

### 图片加载逻辑
场景图片加载在 `script.js` 的 `updateSceneImage()` 函数中实现：

1. **检查映射**: 首先检查 `sceneImageMap` 中是否有该位置ID的映射
2. **检查文件存在性**: 检查 `existingImages` 列表中图片文件是否实际存在
3. **加载图片**: 如果文件存在，创建Image对象加载图片
4. **显示占位符**: 如果文件不存在或加载失败，显示文字占位符

### 代码位置
- **图片映射表**: `script.js` 第869-879行 (`sceneImageMap`)
- **文件存在性检查**: `script.js` 第881-889行 (`existingImages`)
- **占位符描述**: `script.js` 第913-923行 (`sceneDescriptions`)
- **图片加载函数**: `script.js` 第867-906行 (`updateSceneImage()`)
- **占位符显示函数**: `script.js` 第911-930行 (`showSceneImagePlaceholder()`)

## 添加新图片的步骤

### 步骤1: 创建图片文件
1. 使用图像编辑软件创建场景图片
2. 建议尺寸: 1920x1080 像素 (16:9 比例)
3. 保存为PNG格式，优化文件大小
4. 将文件保存到 `assets/scenes/` 目录
5. 文件名应与 `sceneImageMap` 中的映射一致

### 步骤2: 更新代码
1. 在 `existingImages` 对象中添加新图片的条目
   ```javascript
   const existingImages = {
       'college_dorm.png': true,
       'new_scene.png': true  // 添加这一行
   };
   ```

### 步骤3: 测试图片加载
1. 打开游戏，导航到对应位置
2. 检查控制台是否有加载成功消息
3. 验证图片是否正确显示

## 图片规范

### 文件格式
- **格式**: PNG (支持透明背景)
- **色彩模式**: RGB
- **压缩**: 使用无损或高质量压缩

### 尺寸规范
- **主尺寸**: 1920x1080 像素 (16:9)
- **备用尺寸**: 也可以使用其他16:9比例
- **最小尺寸**: 1280x720 像素

### 内容要求
1. **风格一致**: 所有图片应保持相似的视觉风格
2. **主题匹配**: 图片内容应与场景描述一致
3. **氛围营造**: 图片应增强游戏的氛围和沉浸感
4. **可读性**: 文字和重要元素应清晰可辨

## 故障排除

### 常见问题

#### 问题1: 图片不显示
**可能原因**:
1. 文件名拼写错误
2. 文件路径不正确
3. 图片文件损坏
4. `existingImages` 中未添加条目

**解决方案**:
1. 检查控制台错误信息
2. 验证文件路径和文件名
3. 检查图片文件是否能正常打开
4. 确保在 `existingImages` 中添加了条目

#### 问题2: 图片加载慢
**可能原因**:
1. 图片文件过大
2. 网络连接问题
3. 服务器响应慢

**解决方案**:
1. 优化图片文件大小 (目标: <500KB)
2. 使用图片压缩工具
3. 考虑使用WebP格式 (如果支持)

#### 问题3: 占位符显示不正确
**可能原因**:
1. `sceneDescriptions` 中缺少描述
2. CSS样式问题
3. 字体图标未加载

**解决方案**:
1. 在 `sceneDescriptions` 中添加位置描述
2. 检查CSS控制台错误
3. 验证Font Awesome是否正确加载
