# 《悬观谜案：百年轮回》文字交互式解密游戏

## 项目概述

《悬观谜案：百年轮回》是一款中式恐怖悬疑风格的文字交互式解密游戏。玩家将扮演土木工程专业学生林墨，接受发小陈青山的邀请，与女友苏晓雨前往西南山区参观悬云观，在探索过程中遭遇一系列超自然事件，揭开道观百年轮回的秘密。

**游戏类型**：文字交互式解密游戏<br>
**主题风格**：中式恐怖悬疑<br>
**核心玩法**：探索、解谜、对话、收集<br>
**技术栈**：HTML5 + CSS3 + JavaScript (ES6+)<br>
**数据格式**：JSON<br>
**框架依赖**：无外部框架，纯前端实现

## 功能模块设计

### 1. 核心游戏类 (AdventureGame)

位于 `script.js` 中的核心控制器类，负责管理整个游戏的生命周期。

#### 主要功能：
- **游戏状态管理**：玩家状态、世界状态、游戏数据
- **数据加载系统**：支持JSON数据文件和硬编码数据两种方式
- **场景渲染系统**：动态渲染场景描述和交互元素
- **命令处理系统**：解析和执行玩家输入的命令
- **交互系统**：处理点击交互和命令交互

#### 状态管理结构：
```javascript
state = {
    player: {
        name: "林墨",
        location: "college_dorm",
        inventory: [],           // 道具库存
        clues: [],              // 收集的线索
        sanity: 100,            // 理智值（0-100）
        health: 100,            // 健康值（0-100）
        discoveredLocations: Set, // 已探索位置
        currentQuest: null,     // 当前任务
        quests: {               // 任务系统
            active: [],         // 进行中任务
            completed: [],      // 已完成任务
            failed: []          // 失败任务
        },
        flags: Map,             // 游戏标志位
        relationships: Map      // NPC关系系统
    },
    world: {
        time: "15:00",          // 游戏内时间
        date: "2025-01-15",     // 游戏内日期
        weather: "snowstorm",   // 天气系统
        chapter: 1,             // 当前章节
        triggeredEvents: Set    // 已触发事件
    }
}
```

### 2. UI界面系统

基于 `index.html` 的现代响应式布局，包含以下主要界面组件：

#### 顶部标题栏
- 游戏标题和章节信息
- 时间、天气、理智值实时显示
- 中式恐怖风格配色

#### 主游戏区域
- **场景描述区**：动态显示当前位置描述和交互元素
- **操作记录区**：显示玩家行动反馈和历史记录
- **命令输入区**：支持文本输入命令或点击高亮文字交互

#### 右侧信息面板
- **快捷按钮**：道具、地图、笔记、任务、保存、加载、设置、帮助
- **道具库存面板**：网格化道具展示，支持查看、使用、丢弃、组合
- **任务面板**：任务列表和进度追踪
- **地图面板**：道观地图展示，已探索/未解锁位置标记
- **笔记面板**：线索收集和分类整理

#### 底部状态栏
- 玩家信息（姓名、职业）
- 游戏进度显示
- 导航方向按钮（北、西、东、南）
- 系统信息（版本、自动保存状态）

#### 模态对话框系统
- **对话模态框**：NPC对话交互，支持分支选项
- **提示模态框**：游戏提示和系统消息
- **道具详情模态框**：道具详细信息展示
- **加载遮罩**：游戏加载动画（阴阳鱼主题）

### 3. 数据管理系统

#### 位置数据模型 (`data/chapter1.json`)
```json
{
    "id": "location_id",
    "name": "位置名称",
    "description": "位置详细描述",
    "interactives": [
        {
            "id": "interactive_id",
            "name": "交互对象名称",
            "type": "examine|npc|item|device",
            "description": "交互对象描述",
            "actions": ["examine", "talk", "take", "use"],
            "result": {
                "clue": "线索ID",       // 可选的线索奖励
                "item": "道具ID",       // 可获得的道具
                "feedback": "操作反馈", // 操作后的文本反馈
                "dialogue": "对话ID"    // NPC对话ID
            }
        }
    ],
    "exits": [
        {
            "direction": "方向",
            "target": "目标位置ID",
            "description": "出口描述",
            "locked": false
        }
    ],
    "characters": ["npc_id列表"]
}
```

#### 交互元素类型：
- **蓝色** (`--color-interact-blue`): 可检查对象
- **绿色** (`--color-interact-green`): 可收集道具
- **黄色** (`--color-interact-yellow`): 重要线索
- **红色** (`--color-interact-red`): 危险/机关
- **紫色** (`--color-interact-purple`): NPC角色
- **橙色** (`--color-interact-orange`): 可互动设备

### 4. 命令处理系统

支持中英文命令输入，主要命令类型：

#### 基本命令
- `查看/检查/examine [对象]` - 检查场景中的对象
- `前往/go [方向/位置]` - 移动到指定位置
- `使用/use [道具] [目标]` - 使用道具
- `拿起/take [道具]` - 拾取道具
- `丢弃/drop [道具]` - 丢弃道具
- `交谈/talk [NPC]` - 与NPC对话
- `组合/combine [道具1] [道具2]` - 组合道具

#### 系统命令
- `保存/save` - 保存游戏进度
- `加载/load` - 加载游戏存档
- `帮助/help` - 显示帮助信息
- `清空/clear` - 清空操作记录
- `状态/status` - 查看玩家状态

### 5. 游戏机制系统

#### 道具系统
- **道具分类**：工具、线索、钥匙、消耗品、特殊物品
- **库存管理**：12格库存限制，网格化展示
- **道具交互**：使用、查看、丢弃、组合
- **道具效果**：解锁区域、解谜、恢复状态、触发事件

#### 线索系统
- **自动收集**：发现线索自动记录在调查笔记中
- **线索分类**：人物、地点、事件、物品、背景
- **推理整合**：相关线索自动关联，揭示隐藏信息
- **进度推进**：收集足够线索解锁新对话和区域

#### 任务系统
- **任务类型**：主线任务、支线任务、隐藏任务
- **任务结构**：多步骤任务链，步骤间依赖关系
- **任务追踪**：实时显示任务目标和进度
- **任务奖励**：道具、线索、新区域解锁、剧情推进

#### 对话系统
- **分支对话**：对话选项影响NPC关系和剧情走向
- **关系系统**：NPC好感度影响对话内容和帮助程度
- **上下文对话**：基于玩家进度和线索的智能对话响应
- **对话记忆**：NPC记住之前对话内容，提供连贯体验

#### 状态管理系统
- **理智值系统**：遭遇恐怖事件降低理智值，影响判断力
- **健康值系统**：受伤或中毒降低健康值，影响行动能力
- **时间系统**：游戏内时间流逝，影响事件触发和NPC行为
- **天气系统**：动态天气变化，影响探索难度和事件发生

## 游戏玩法

### 核心玩法循环
1. **探索场景**：查看当前位置描述和交互元素
2. **发现线索**：点击高亮文字或输入命令与场景交互
3. **收集道具**：拾取和使用道具解决谜题
4. **推进剧情**：完成任务，解锁新区域和剧情
5. **记录推理**：在调查笔记中整理线索，发现真相

### 解谜机制
- **工程学解谜**：利用主角土木工程专业背景解决机关
- **逻辑推理**：基于收集的线索进行逻辑推理
- **道具组合**：组合不同道具产生新效果
- **环境互动**：利用环境元素解决谜题

### 多结局系统
玩家的选择影响：
1. **剧情走向**：不同对话选择导向不同分支
2. **角色关系**：NPC好感度影响结局
3. **线索收集**：是否发现关键线索影响结局真相
4. **道德选择**：伦理困境影响最终结局

## 安装与运行

### 本地运行
1. 下载项目文件到本地目录
2. 打开 `游戏/index.html` 文件
3. 使用现代浏览器（Chrome 90+ / Firefox 88+ / Edge 90+）打开
4. 游戏将自动加载并开始

### 注意事项
- 需要现代浏览器支持ES6+特性
- 需要网络连接加载Google Fonts和Font Awesome
- 支持离线运行（需提前缓存外部资源）
- 不支持IE浏览器

## 开发指南

### 扩展新章节
1. 在 `data/` 目录下创建 `chapterX.json` 文件
2. 遵循现有JSON数据结构格式
3. 在 `script.js` 中更新章节加载逻辑
4. 添加新章节的UI显示支持

### 添加新功能模块
1. 在 `AdventureGame` 类中添加新方法
2. 在 `index.html` 中添加对应的UI元素
3. 在 `style.css` 中添加样式定义
4. 在 `initUIEvents()` 中绑定事件处理

### 数据格式规范
- **位置ID**：使用小写蛇形命名（如 `college_dorm`）
- **道具ID**：前缀 `item_`（如 `item_tools`）
- **线索ID**：前缀 `clue_`（如 `clue_engineering_background`）
- **NPC ID**：使用拼音或英文名（如 `su_xiaoyu`）
- **对话ID**：前缀 `dialogue_`（如 `dialogue_suxiaoyu_subway`）

### 调试工具
- 浏览器开发者工具控制台查看游戏状态
- `console.log` 输出调试信息
- 游戏内自带系统反馈信息
- 错误处理和用户友好提示
