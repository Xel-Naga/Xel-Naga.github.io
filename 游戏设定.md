# 《悬观谜案：百年轮回》文字交互式解密游戏系统设计

## 一、游戏概述

### 游戏类型
基于文本的交互式推理解密游戏，玩家扮演主角林墨（土木工程专业学生），在暴雪困观的悬云观中调查命案、破解机关、揭开百年秘密。

### 核心玩法
- **文字探索**：阅读环境描述，发现线索
- **交互解谜**：点击高亮文字检查/收集/使用物品
- **道具管理**：收集和使用道具解决机关谜题
- **任务推进**：完成关键动作解锁新区域和剧情
- **对话选择**：与NPC对话获取信息和线索

### 技术架构
- **前端**：Web-based文字游戏界面（HTML/CSS/JavaScript）
- **后端**：游戏状态管理、逻辑处理（可选Node.js/Python）
- **数据存储**：JSON格式的游戏数据（场景、道具、任务、对话）

```
└─────────────────────────────────────────┘
```

#### 交互功能
1. **道具查看**：点击道具显示详细信息
2. **道具使用**：
   - 在场景中选择使用目标（门、机关、NPC）
   - 系统验证使用条件
   - 执行使用效果
3. **道具组合**：
   - 选择两个可组合道具
   - 合成新道具（如：齿轮碎片+润滑油=完整齿轮）
4. **道具丢弃**：
   - 从库存移除
   - 可能影响后续解谜
5. **道具限制**：
   - 最大携带数量
   - 部分道具不可丢弃（关键剧情物品）

### 3.4 道具获取方式
1. **初始携带**：主角开场道具
2. **场景收集**：点击环境中的可收集物品
3. **NPC给予**：完成任务后获得
4. **机关奖励**：破解谜题后获得
5. **组合合成**：多个道具组合成新道具

```javascript
// 点击响应逻辑
function handleHighlightClick(element) {
  const type = element.dataset.type;
  const id = element.dataset.id;

  switch(type) {
    case 'examine':
      // 显示详细检查信息
      showExaminationDetail(id);
      break;
    case 'collect':
      // 收集物品到库存
      collectItem(id);
      break;
    case 'clue':
      // 记录线索到笔记本
      addClueToNotebook(id);
      break;
    case 'danger':
      // 警告或触发事件
      handleDanger(id);
      break;
    case 'npc':
      // 开始对话
      startDialogue(id);
      break;
    case 'interact':
      // 交互操作
      interactWithObject(id);
      break;
  }
}
```

```
1. 游戏开始
   → 显示第二章开场剧情
   → 玩家获得控制权

2. 山门场景
   → 阅读环境描述（带颜色标注）
   → 点击[蓝色]石狮检查
   → 获得线索：底座有新鲜摩擦痕迹
   → 点击[黄色]匾额查看
   → 记录线索："悬云"藏头
   → 点击[紫色]明心道士对话
   → 了解道观基本信息
   → 点击[橙色]山门进入前院

3. 任务触发
   → 进入前院触发任务"调查道观异常"
   → 任务列表更新

4. 解谜过程
   → 前往三清殿
   → 点击[红色]倾斜地板
   → 系统提示：需要使用水平仪测量
   → 打开道具栏选择水平仪
   → 对地板使用水平仪
   → 获得测量结果：2度倾斜

   → 完成所有调查步骤
```
